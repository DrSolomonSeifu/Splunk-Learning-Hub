Section 6.1

Saving Searches as Reports

What Is a Report?

A report in Splunk is a saved search that can include statistics or visualizations. Once saved, it can be re-run anytime to get the latest results, shared with teammates, and added to dashboards.

Four Ways to Create a Report

MethodHow
From SearchRun a search → Save As → Report
From PivotUse the Pivot drag-and-drop tool → Save as report
From SettingsSettings → Searches, Reports, and Alerts → New Report
From Dashboard PanelConvert an inline search panel into a report

Saving a Report — Required Fields

  • Title (required) — Use letters, numbers, underscores, or dashes
  • Description (optional) — Explains the purpose
  • Time Range Picker (optional) — Lets viewers re-run with different time ranges
🧪 Lab 1

Save Your First Report

  1. Run this search:
    SPL
    index=_internal sourcetype=splunkd | stats count by log_level
  2. Click Save As → Report
  3. Title: Log_Level_Summary
  4. Add a description: "Counts events by log level"
  5. Check "Time Range Picker" and click Save
  6. Explore the Report Viewing Page — note the Edit, Add to Dashboard, and Export options
✏️
Section 6.2

Editing Reports & Permissions

Post-Save Settings

SettingPurpose
PermissionsControl who can view (Read) or modify (Write)
ScheduleRun automatically at set intervals
AccelerationPre-process data for faster results on large datasets
EmbedGenerate HTML to embed the report externally

Permissions Model

ScopeWho Can See It
Private (default)Only the creator
AppUsers within the same Splunk app
All AppsEveryone across the entire Splunk instance

Run As: Choose whether the report runs using the Owner's permissions (viewers see data they may not normally access) or the User's permissions (safer, enforces role-based restrictions).

🧪 Lab 2

Edit Permissions on Your Report

  1. Navigate to Reports from the top menu
  2. Find your Log_Level_Summary report
  3. Click Edit → Edit Permissions
  4. Change from Private to App visibility
  5. Set Read access for the user role
  6. Click Save

Section 6.3

Scheduling Reports

What Is a Scheduled Report?

A scheduled report runs automatically at set times (hourly, daily, weekly, or custom Cron). Each run can trigger up to four actions.

ActionWhat It Does
Send EmailEmail results to recipients
Output to LookupSave results as a CSV file
WebhookNotify external systems (Slack, Teams, ServiceNow)
Log & IndexIndex the results for future searches

Cron Schedule Format

For custom timing, use a Cron expression with 5 positions:

PositionFieldRange
1Minute0–59
2Hour0–23
3Day of Month1–31
4Month1–12
5Day of Week0–6 (0 = Sunday)

Example: 0 8 * * 1 = Run at 8:00 AM every Monday

⚠️ Important

Once a report is scheduled, the time range picker is removed from the report view. The report always uses the time range set in the schedule.

🧪 Lab 3

Schedule a Report

  1. Open your Log_Level_Summary report
  2. Click Edit → Edit Schedule
  3. Check Schedule Report
  4. Set to Run daily
  5. Set Time Range to Last 24 hours
  6. Set Schedule Window to 15 minutes
  7. Explore the Add Actions dropdown (don't add one yet)
  8. Click Save

Section 6.4

Reports with Tables

Creating Tabular Reports

Reports that use transforming commands (stats, top, rare, table, addcoltotals) automatically display as statistics tables. These are the structured, data-driven reports used most in professional environments.

🧪 Lab 4

Build a Table Report

  1. Run this search:
    SPL
    index=_internal sourcetype=splunkd
| stats count as "Events", dc(component) as "Components", dc(host) as "Hosts" by log_level
| sort - Events
| addcoltotals
  2. Review the Statistics tab — note the total row at the bottom
  3. Click Save As → Report
  4. Title: Event_Summary_by_LogLevel
  5. Save and view the Report Viewing Page
📈
Section 6.5

Reports with Charts

Chart Types in Splunk

Chart TypeBest ForData Series
PieProportions of a wholeSingle series
Bar / ColumnComparing categoriesSingle or multiple
LineTrends over timeMultiple series
AreaStacked values over timeMultiple series
ScatterCorrelation between two variablesTwo series
BubbleThree-dimensional comparisonThree series

Advanced options: Single Value, Radial Gauge, Filler Gauge, Cluster Map

Saving Content Types

When saving a chart report, choose what to include:

  • Chart + Statistics Table — both the visualization and raw data
  • Chart Only — saves just the visual
  • Statistics Table Only — saves just the tabular data
🧪 Lab 5

Create a Chart Report

  1. Run a timechart search:
    SPL
    index=_internal sourcetype=splunkd | timechart count by log_level
  2. Click the Visualization tab
  3. Try different chart types: Line → Column → Area → Pie
  4. Click Format → experiment with Stack Mode, Legend position
  5. Click Save As → Report
  6. Title: LogLevel_Trend_Chart
  7. Select content type: Column Chart and Statistics Table
  8. Save and review

Section 6.6

Creating Dashboards

What Is a Dashboard?

A dashboard is a visual interface made up of panels. Each panel shows a chart, table, search result, or input control. Panels are usually powered by saved reports or inline searches.

Two Dashboard Frameworks

FrameworkCodeNotes
Classic DashboardsSimple XMLTraditional, simpler — this lesson's focus
Dashboard StudioJSONModern, more flexible (not covered here)
Naming Convention

Splunk recommends: Group_Object_Description

Example: Security_Dashboard_Daily_Alerts

  • Group = Business unit (Security, Sales, IT)
  • Object = Content type (Dashboard, Report)
  • Description = Purpose (Daily_Alerts, Weekly_Traffic)
🧪 Lab 6

Create Your First Dashboard

  1. Click Dashboards from the top menu
  2. Click Create New Dashboard
  3. Title: Training_Dashboard_System_Health
  4. Description: "System health overview for CySA+ training"
  5. Permissions: Private
  6. Type: Classic Dashboards
  7. Click Create — you'll land in the Dashboard Editor
➕
Section 6.7

Adding Reports to Dashboards

Four Ways to Add Panels

MethodDescription
NewBuild a panel from scratch with a fresh search
New from ReportUse an existing saved report
Clone from DashboardCopy a panel from another dashboard
Prebuilt PanelInsert ready-made panels from Splunk apps

Report vs Inline Search

Panel TypeBehavior
Powered by ReportStays linked — changes to the report update the panel automatically
Inline SearchCopies the SPL — panel is independent, won't change if the original report changes
🧪 Lab 7

Build a Multi-Panel Dashboard

  1. In your Training_Dashboard_System_Health, click + Add Panel
  2. Panel 1 — From Report: Click "New from Report" → select your Event_Summary_by_LogLevel report → Add to Dashboard
  3. Panel 2 — From Report: Click "New from Report" → select your LogLevel_Trend_Chart report → Add to Dashboard
  4. Panel 3 — New Inline: Click "New" → choose "Column Chart" → enter search:
    SPL
    index=_internal sourcetype=splunkd | top limit=5 component
    Set Time Range to "Last 24 hours", Title: "Top 5 Components" → Add to Dashboard
  5. Click the green Save button
🔧
Section 6.8

Editing Dashboards

DrilldownsMake panels clickable — link to detailed searches or other dashboards

Export Options

WhatFormats
Entire DashboardPDF only

🧪 Lab 8

Customize Your Dashboard

  1. Click Edit on your dashboard
  2. Rearrange: Drag the chart panel to the top
  3. Resize: Make the chart panel wider
  4. Click a panel's pencil icon → change the visualization type (try switching to a pie chart)
  5. Click Save
  6. Test Export → export the dashboard as PDF
  7. Hover over a chart panel → export its data as CSV

Knowledge Check

Lesson 200.6 Review

1. What type of search can be saved as a report?▼
A search with transforming commands (stats, chart, timechart, top, rare).
2. What must be added to display statistical data in a report?▼
A transforming command such as stats, chart, or timechart.
3. What options are available after saving a report?▼
Edit title, description, permissions, schedule, acceleration, and embed settings.
4. What are the permission options for a report?▼
Scope: App or All Apps. Run As: Owner or User. Roles: Read or Write access per role.
5. What actions can a scheduled report trigger?▼
Send email, output to lookup (CSV), run a webhook, log and index data.
6. What is an advantage of scheduling a report?▼
Runs automatically without manual intervention and can trigger actions like email notifications.
7. What visualization types are available?▼
Column, bar, pie, line, area, scatter, bubble, single value, radial gauge, cluster map.
8. What are two ways to create a dashboard?▼
From the Search & Reporting app (Save As → Dashboard) or from the Dashboard Editor (Dashboards → Create New).
9. What is Splunk's recommended dashboard naming convention?▼
Group_Object_Description. Example: Security_Dashboard_DailyAlerts
10. What are the panel editing options?▼
Add/remove panels, resize, rearrange, change search query, time range, visualization type, and refresh rate.
