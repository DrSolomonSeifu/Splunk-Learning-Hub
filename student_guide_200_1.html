Introduction

Welcome to Splunk

What Is Splunk, and Why Are We Learning It?

Splunk is a platform that helps organizations search, analyze, and visualize machine-generated data. It ingests data from many different sources and provides real-time insights so teams can quickly spot trends and anomalies.

With its visualization tools and scalability, Splunk supports a wide range of use cases — IT operations, security and compliance, and business analytics.

As an IT or security professional, you need to monitor and troubleshoot many systems, applications, and infrastructure components. Splunk gives you visibility into all of them by ingesting and analyzing the logs and metrics they generate — all from one place.

Key Concept — One Pane of Glass

Instead of logging into ten different servers to read ten different log files, Splunk pulls all of that data into one searchable place. Think of it as a search engine — like Google — but for your organization's own machine data.

How This Lesson Flows

We start with the raw material (machine data), then meet the tool that tames it (Splunk), look under the hood (components), see what it's used for (use cases), learn how it's extended (apps) and installed (deployment types), and finish hands-on: navigating Splunk and customizing your settings.

Section 1.1

Machine Data

What Is Machine Data?

Machine data is any type of data that is generated by machines, systems, or devices — not typed by a human.

It is produced automatically, continuously, and in large quantities. That makes it difficult to manage and analyze without the right tools. A single busy web server can generate millions of log lines per day — no human could read them all.

Key Concept — Digital Exhaust

Every machine leaves a trail of records about what it is doing — like footprints in the snow. Every login, error, connection, and update leaves a line in a log somewhere. Machine data is that trail, and it tells the story of what actually happened on your systems.

The Six Main Types of Machine Data

TypeWhat It ContainsExample
Server logsServer events: user logins, system errors, software updatesLinux /var/log/auth.log recording a failed SSH login
Network logsNetwork traffic: source and destination IP addresses, protocols used, data transfer ratesA firewall log showing traffic from 10.0.0.5 to port 443
Sensor dataReadings from sensors embedded in machines or devices: temperature, humidity, pressure, other environmental conditionsA data-center sensor reporting rack temperature every 30 seconds
Application logsApplication events: user interactions, errors, performance metricsA web app logging "checkout completed in 1.2s"
Security dataSecurity events: login attempts, system breaches, virus/malware detectionsAntivirus logging "malware quarantined on HOST-42"
IoT dataData from Internet of Things devices: smart homes, wearables, industrial sensorsA smart thermostat reporting usage patterns

What Raw Machine Data Actually Looks Like

Here is a single, real-looking log event. Notice how much information is packed into one messy line:

Raw Log Event
Jul 04 09:15:22 web-server-01 sshd[2841]: Failed password for admin from 203.0.113.45 port 52144 ssh2

A trained eye can spot a timestamp, a hostname, a process, a failed login, a username, and a source IP address. Now imagine millions of lines like this arriving every hour, in dozens of different formats. That is why raw machine data needs a tool like Splunk.

⚠️ Common Mistake

Beginners often assume machine data means only "server logs." It is much broader — anything a machine records automatically counts, from a firewall entry to a smartwatch heartbeat reading.

📋 Section 1.1 — Key Takeaways

  • Machine data = data generated automatically by machines, systems, or devices
  • Types include server logs, network logs, sensor data, application logs, security data, and IoT data
  • It is a valuable source of insight — it can optimize operations, improve security, and drive business decisions
  • Raw machine data is huge, complex, and not human-friendly — which is exactly where Splunk enters the stage

Section 1.2

Splunk

What Is Splunk?

Splunk is a powerful platform that helps organizations search, analyze, and visualize machine-generated data.

It provides a scalable, real-time approach to ingesting and processing large volumes of data from a wide range of sources — servers, applications, network devices, and security systems.

Splunk's indexing engine makes data easy to search, correlate, and analyze, giving IT professionals a comprehensive view of their environment.

What Else Does Splunk Offer?

  • Pre-built apps and add-ons — packages that help you get even more value from machine data, such as monitoring security events, analyzing network traffic, and managing IT operations (covered in Section 1.5)
  • Visualization tools — charts, graphs, and dashboards that help organizations quickly identify trends, anomalies, and potential problems
  • Machine learning capabilities — Splunk can detect anomalies and even predict future issues before they happen
Key Concept — Why "Real Time" Matters

Traditional log review happens after something breaks. Splunk analyzes data as it arrives, so you can catch a spreading attack or a failing server while it is happening — not the next morning.

The Five Main Functions of Splunk

Splunk operates through five main functions. Learn these in order — each one builds on the previous:

#FunctionIn One Sentence
1Index DataIngest data and store it in a searchable format
2Search and InvestigateQuery and filter data in real time
3Add KnowledgeEnrich data with custom fields, tags, and lookups
4Monitor and AlertWatch data continuously and get notified when conditions are met
5Report and AnalyzeTurn data into reports, dashboards, and decisions

Function 1 — Index Data

Splunk indexing is the process of ingesting data from various sources — logs, metrics, and configuration files — and storing it in a format that can be easily searched, correlated, and analyzed.

The Splunk indexing engine uses a multi-step process:

  1. Break the data into events — each log entry becomes its own searchable record
  2. Parse the events to extract fields — pull out useful pieces like timestamps, IPs, and usernames
  3. Normalize the fields into a common format — so different data sources can be compared side by side
  4. Store the normalized events in an index — Splunk's searchable "database"

Analogy: think of a librarian receiving a truckload of loose papers, sorting them into labeled folders, and filing them in the right drawers — so anyone can find any page in seconds.

Function 2 — Search and Investigate

Search and Investigate lets users search, filter, and investigate data in real time. You can build complex queries and filters to extract insights from your data.

Key capabilities include:

  • Real-time searching — query data as it streams in
  • Search Processing Language (SPL) — Splunk's query language for creating complex search queries (the focus of Lesson 200.2)
  • Field extraction — pull structured fields out of raw text
  • Visualization — view results as charts and graphs
  • Correlation — connect related events across different systems

Function 3 — Add Knowledge

Add Knowledge means adding custom knowledge to the data already indexed in Splunk. You can create custom fields, tags, event types, and other data structures that help you better understand and analyze your data.

  • Custom fields — extract new fields from raw data that Splunk did not find automatically
  • Tags — apply labels to events to group related events together (for example, tag all authentication events as login)
  • Lookups — match values in your data to other data sources to add context (for example, turn error code 404 into "Page Not Found," or an employee ID into a name and department)

Function 4 — Monitor and Alert

Monitor and Alert helps users proactively monitor their data and receive alerts when certain conditions are met — so you find problems before your users (or attackers) do.

  • Real-time monitoring — watch your data continuously as it arrives
  • Custom alerts — trigger on specific conditions, such as "more than 10 failed logins in 5 minutes"
  • Alert actions — define what happens when an alert fires: send an email, run a script, create a ticket
  • Alert escalation — define escalation paths so alerts are addressed in a timely manner even if the first responder misses them

Function 5 — Report and Analyze

Report and Analyze helps users analyze and visualize their data to gain insights into their environment. You can create reports and dashboards that give a comprehensive view of your data, make informed decisions, and present and share findings on a single pane of glass.

  • Data visualization — a range of tools including charts, graphs, and dashboards
  • Custom reports — build reports tailored to exactly the question you need answered
  • Scheduled reports — schedule reports to run automatically at specific intervals (for example, a compliance report every Monday at 8 AM)
  • Dashboards — combine multiple visualizations into one live view for your team or management

📋 Section 1.2 — Key Takeaways

  • Splunk lets organizations collect, index, search, and analyze machine-generated data from virtually any source, in real time
  • The five functions: Index Data → Search & Investigate → Add Knowledge → Monitor & Alert → Report & Analyze
  • Its user-friendly interface and advanced features make it a popular tool for IT professionals to monitor, troubleshoot, and optimize systems and infrastructure

Section 1.3

Splunk Components

The Three Core Components

Splunk Enterprise is Splunk's core product — it provides the indexing and search capabilities needed to process machine data. It is composed of several components that work together to process, index, and analyze that data:

ComponentRoleThink of It As…
ForwardersCollect data at the source and send it to indexersDelivery trucks bringing the data in
IndexersReceive, process, and index data, making it searchableLibrarians organizing everything on the shelves
Search HeadsLet users search, analyze, and visualize the dataThe front desk where you ask your questions

Data flows in one direction: Forwarders → Indexers → Search Heads → You.

Indexers

Indexers receive, process, and index data from various sources, making it searchable and available for analysis. They allow data to be stored and retrieved quickly and efficiently — even across massive datasets.

In larger environments, multiple indexers can be clustered together to provide high availability and improved performance.

The index itself is composed of several files:

  • Raw data files — the original data exactly as it was ingested
  • Metadata files — information about each event, such as its timestamp and source
  • Index files — the lookup structures that make searching fast
Key Concept — Buckets

Splunk uses a process called "bucketing" to manage index files. The index is divided into time-based buckets that group together events from a specific time period — such as one hour or one day. When you search "last 24 hours," Splunk only opens the buckets from that window, which is a big part of why searches are fast.

Search Heads

The search head is the component that enables users to search and analyze data in real time. When you run a search query, it goes to the search head, which distributes the search to one or more indexers.

The search head then receives the results back from the indexers and presents them to you in a user-friendly web interface, where you can visualize, analyze, and act upon the data.

The search head is also where you create alerts and reports.

Forwarders

The forwarder is responsible for collecting and forwarding data from various sources to the Splunk indexers. It runs on the source machine — the server, workstation, or device that is generating the data — and collects logs, metrics, and other data in real time, then forwards it to one or more indexers for processing.

There are two types:

TypeDescriptionWhen to Use
Universal ForwarderA lightweight agent that can be deployed on a wide range of machines to collect data and forward it to SplunkThe standard choice — minimal footprint, just collects and sends
Heavy ForwarderA full Splunk Enterprise instance that can index, search, and change (transform) data, as well as forward itWhen data must be filtered, masked, or routed before it reaches the indexers
⚠️ Remember

Universal = lightweight, forward only. Heavy = full Splunk instance that can also process data. This distinction appears in interviews and certification questions constantly.

📋 Section 1.3 — Key Takeaways

  • Indexers, Search Heads, and Forwarders are the core components of the Splunk platform
  • Indexers parse and timestamp raw data, creating index files organized into time-based buckets
  • The Search Head is your web-based interface for searching and analyzing data
  • Forwarders are agents that run on the machines generating the data and forward it to the indexers
  • Working together, these components let Splunk efficiently process and analyze machine data at scale

Section 1.4

Uses of Splunk

Where Is Splunk Used?

Splunk is a powerful platform used across industries for a wide range of use cases:

Use CaseWhat Splunk DoesExample
IT OperationsMonitor and troubleshoot IT infrastructure, applications, and services so teams quickly identify and resolve issues that impact business operations. Also used for performance monitoring, capacity planning, and log analysis.Finding why a website slowed down at 2 PM by correlating server and application logs
SecurityDetect and respond to security threats in real time — identify and investigate suspicious activity, conduct forensic analysis, and manage security incidents. Also used for compliance reporting and auditing.A SOC analyst spotting a burst of failed logins followed by a successful one
Business AnalyticsAnalyze customer behavior, marketing campaigns, and product performance for data-driven decisions and better customer engagement. Also used for financial analysis, supply chain management, and risk management.Tracking which product pages customers abandon before checkout
Internet of Things (IoT)Collect and analyze data from IoT devices and sensors to monitor and optimize IoT deployments, improve product quality, and enhance customer experiences.A factory monitoring sensor data to predict machine failures before they happen
Key Concept — Same Data, Different Questions

Notice that all four use cases search the same kind of machine data — they just ask different questions. IT asks "is it healthy?", Security asks "is it safe?", Business asks "is it profitable?", IoT asks "is it performing?". One platform, many audiences.

Use Case Walkthrough — Resolving a Network Security Breach

The situation: A company is alerted to a potential security breach on its network. The IT security team needs to quickly identify the source of the breach, assess the extent of the damage, and take action to contain and remediate the issue.

The investigation: Using Splunk's search and investigation capabilities, the team quickly searches for suspicious activity and correlates events across multiple systems. Splunk's dashboards and reports help them visualize and analyze network traffic, identify anomalies and patterns, and gain insight into the attack.

The finding: The breach was caused by a phishing attack that compromised a user account. The team can see that the attacker used the compromised account to access sensitive data on the network and attempted to exfiltrate (steal out) the data using a command and control (C2) server.

The response: Armed with this evidence, the team responds quickly — disabling the compromised user account and blocking the attacker's IP address.

⚠️ Why This Matters to You

This walkthrough is a preview of your future job. In a SOC, this exact sequence — detect → investigate → identify → contain → remediate — happens inside Splunk every day. Everything you learn in this course builds toward being able to do this yourself.

📋 Section 1.4 — Key Takeaways

  • IT operations: monitoring and troubleshooting servers, networks, applications, and databases
  • Security & compliance: detecting and responding to threats, investigating incidents, ensuring regulatory compliance
  • Business analytics: analyzing data and providing insights to drive decision-making and improve performance
  • IoT: monitoring and optimizing device and sensor deployments
  • With its ability to index, search, and analyze machine data from virtually any source, Splunk provides value to many industries and use cases

Section 1.5

Splunk Apps

What Are Splunk Apps?

Splunk apps are pre-built packages of dashboards, reports, and other content that provide specific functionality on top of the core Splunk platform.

They are designed to help users quickly solve specific use cases without needing the expertise to build custom dashboards and reports from scratch. In short: apps extend Splunk's capabilities.

Think of them exactly like apps on your phone: your phone works without them, but apps make it useful for your specific needs.

Who Makes Apps — and Which Ones Can You Trust?

  • Apps can be created by Splunk itself, by third-party vendors, or by individuals — you can even build your own using the Splunk app framework
  • Apps marked with the Splunk Built logo are made by Splunk
  • Apps marked with the Splunk Certified logo have been reviewed and certified by Splunk to meet its quality standards
  • An app can still provide plenty of value even without being certified by Splunk — certification is a quality signal, not a requirement

Under the Hood — What an App Actually Is

An app is not magic — a Splunk app is simply a collection of Splunk configuration files stored in a folder. On a Windows machine:

File Path
C:\Program Files\Splunk\etc\apps\[app name]

Every app you install gets its own folder there. Knowing this helps you troubleshoot later — if an app misbehaves, you know exactly where its files live.

Apps vs. Add-Ons

A Splunk add-on is a software module that provides additional functionality to the Splunk platform or to Splunk apps — usually in the form of data inputs, parsers, or other components that allow Splunk to ingest and analyze data from specific sources.

AppAdd-On
What it gives youDashboards, reports, visualizations — things you seeData inputs and parsers — plumbing that works behind the scenes
PurposeSolve a use case out of the boxGet data from a specific source into Splunk correctly
ExampleSplunk App for AWS (ready-made AWS dashboards)Splunk Add-on for AWS (knows how to pull and parse AWS data)

Add-ons are often developed by third-party vendors, are available from the Splunkbase app store, and can be installed and configured using the Splunk Web Interface.

Where to Get Apps

  • Many vendors — such as Cisco, Amazon, and Microsoft — create Splunk apps tailored to help monitor their own products
  • Download apps from within the Splunk interface or from Splunkbase.com
  • Most apps are free; some premium apps require a license

Section 1.6

Splunk Deployment Types

Three Ways to Deploy Splunk

Splunk Enterprise can be deployed in many configurations — from a single instance to distributed deployments of hundreds or thousands of instances, depending on an organization's requirements. There are three primary deployment types:

TypeDescriptionBest For
StandaloneA single Splunk instance performs all functions — data ingestion, indexing, search, and visualizationSmall environments, labs, testing, learning (this is what you use in class)
DistributedMultiple Splunk instances, each responsible for one or more functions — for example, one instance handles data ingestion while another handles indexing and searchMedium to large organizations with high data volumes
CloudSimilar to a distributed deployment, but hosted and managed by a third-party cloud provider such as AWS or Microsoft AzureOrganizations that do not want to manage their own infrastructure or need to scale quickly
Key Concept — Same Components, Different Homes

The deployment type does not change what Splunk does — it changes where the components live. In a standalone deployment, the forwarder role, indexer, and search head all run in one box. In a distributed deployment, each role gets its own dedicated machines. In the cloud, someone else runs those machines for you.

A Small Enterprise Example

A typical small-enterprise distributed deployment looks like this: many universal forwarders installed across servers and workstations, all sending data to a small number of indexers, with one search head where analysts log in and run searches. (The official reference diagrams are at docs.splunk.com.)

Data Flow
[ Forwarders on many machines ]  →  [ Indexers ]  →  [ Search Head ]  →  You

📋 Section 1.6 — Key Takeaways

  • Three primary deployment types: standalone, distributed, and cloud
  • Standalone = one instance does everything; distributed = multiple instances with dedicated roles
  • Cloud = like distributed, but hosted and managed by a third-party provider (AWS, Azure)
  • Organizations choose based on the size and complexity of their environment

Section 1.7

Basic Navigation in Splunk

The Splunk Web Interface

Splunk uses a customizable, web-based interface that gives users access to their indexed and stored data. It includes components such as the search bar, search results page, navigation bar, and dashboard and visualization editors — all designed to be intuitive and efficient for analyzing and understanding data.

Key Concept — Port 8000

You sign into Splunk through a web browser using port 8000. For example: http://localhost:8000 on your own machine, or http://splunk-server:8000 for a remote server. No special client software required — just a browser.

What You Will See After Signing In

AreaWhat It Is
Home page / LauncherYour landing page after login — shows your apps and quick links to get started
App panel / Manage ApplicationsLists installed apps (like Search & Reporting) and lets you browse, install, and manage apps
Splunk barThe black bar across the top — navigation menus for Settings, Activity, Help, and your account
Main user rolesSplunk has three main roles: admin (full control), power (advanced user — can share objects and use more features), and user (standard access)

Section 1.8

Customizing User Settings

Account Settings and Preferences

Splunk lets each user personalize their account and workspace. The two main areas are:

AreaWhat You Can Configure
Account Settings — PersonalFull name, email, and password
Preferences — GlobalTime Zone — ensures timestamps are aligned with you, the user
Preferences — SPL EditorHow the search editor behaves and looks (line numbers, auto-format, editor theme)
Preferences — General / ThemesOverall interface appearance, including light and dark themes
⚠️ The Time Zone Setting Is Not Cosmetic

If your time zone is wrong, every timestamp you see is shifted. Imagine investigating an incident and reporting that an attack happened at 3 AM when it really happened at 8 AM your time — your whole timeline (and your incident report) would be wrong. Set your time zone first, before anything else.

🧪 Lab 4 — Follow Along with the Instructor

Customize Your Account

  1. Click your username in the Splunk bar (top right)
  2. Open Account Settings — review your full name, email, and password options
  3. Open Preferences → Global — set your Time Zone to your local time zone
  4. Open the SPL Editor preferences — note what can be changed
  5. Open General / Themes — try switching between light and dark themes

📋 Section 1.8 — Key Takeaways

  • Account Settings hold your personal info: full name, email, password
  • Preferences include Global (time zone), SPL Editor, and General/Themes
  • Always set the time zone so timestamps align with you — it directly affects investigation accuracy
