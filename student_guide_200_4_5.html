Section 4.1

The Search Pipeline

What Is a Search Pipeline?

A search pipeline is a sequence of commands that process your data step by step. Each command takes the output of the previous one and passes its results to the next — like an assembly line where each station refines the product.

Commands are connected using the pipe symbol |. The first word after each pipe is the command name.

Pipeline Structure
search terms → | command1 → | command2 → | command3 → final results

Anatomy of a Pipeline

SPL
index=web sourcetype=access_combined | stats count by status | sort - count
StageCommandWhat It Does
1index=web sourcetype=access_combinedRetrieves matching events from the index
2stats count by statusGroups by status code and counts each
3sort - countSorts results by count, descending

Step-by-Step Data Flow

Let's trace how data transforms through this pipeline:

SPL
sourcetype=syslog ERROR | top user | fields - percent
StepWhat HappensResult
DiskAll indexed data is stored hereFull data warehouse
Step 1sourcetype=syslog ERROR — retrieves matching eventsFiltered set of syslog error events
Step 2top user — counts frequency per userTop 10 users by error count (+ count & percent columns)
Step 3fields - percent — removes the percent columnClean table: user + count only
Key Concept — Pipeline Flow
  • A pipeline is always read left to right
  • The output of one command becomes the input for the next
  • The first word after each | is the command being used
  • Think in stages: retrieve → filter → transform → present
🧪 Lab 1

Build a Pipeline Step by Step

  1. Run just the base search:
    SPL
    index=_internal sourcetype=splunkd
  2. Add a stats command:
    SPL
    index=_internal sourcetype=splunkd | stats count by component
  3. Add a sort:
    SPL
    index=_internal sourcetype=splunkd | stats count by component | sort - count
  4. Watch how results change at each step — this is the pipeline in action

📋 Section 4.1 Key Takeaways

  • The search pipeline = sequential flow of commands joined by pipes |
  • Each command processes the output of the previous one
  • Start with search terms, then add filter/transform/present commands
  • Data flows left to right, refined at every step

Section 4.2

The fields Command

What Does fields Do?

The fields command includes or excludes specific fields from your search results. It controls which columns appear in your output.

ActionSyntaxExample
Include fieldsfields field1 field2 or fields + field1| fields clientip status
Exclude fieldsfields - field1 field2| fields - password _raw
Wildcardfields user_*Include all fields starting with user_
Exclude internalfields - _*Hides all underscore-prefixed internal fields
⚠️ Syntax Rules
  • Space required between +/- and field names: fields - categoryId ✅ vs fields -categoryId ❌
  • Separate multiple fields with spaces (not commas)
  • At least one field name is required
  • Place fields early in the pipeline for better performance
🧪 Lab 2

Include and Exclude Fields

  1. Include specific fields:
    SPL
    sourcetype=splunkd | fields host component log_level
  2. Exclude a field:
    SPL
    sourcetype=splunkd | fields - _raw _time
  3. Try with NO space after the minus: fields -_raw — what happens?
📝 What happened with no space?

📋 fields Command Key Takeaways

  • fields = field-level control over search results
  • Use + to include, - to exclude (include is default)
  • Always use a space after + or -
  • Place early in pipeline for performance; at the end for clean output

Section 4.2

The dedup Command

What Does dedup Do?

dedup removes duplicate events based on one or more fields. Think of it as cleaning a contact list — keeping only one entry per email address.

SyntaxWhat It Does
dedup userKeeps one event per unique user
dedup 3 clientipKeeps up to 3 events per unique IP
dedup clientip sortby +clientipDedup by IP, keeping lowest IP value
⚠️ Performance Warning

Avoid using dedup on the _raw field — it compares the entire event text, which is extremely slow on large datasets. Dedup on specific, smaller fields instead.

🧪 Lab 3

Dedup in Action

  1. See all values:
    SPL
    index=_internal sourcetype=splunkd | table host component
  2. Remove duplicates:
    SPL
    index=_internal sourcetype=splunkd | table host component | dedup host component
  3. Keep 3 per combination:
    SPL
    index=_internal sourcetype=splunkd | table host component | dedup 3 host component
  4. Compare event counts at each step

📋 dedup Command Key Takeaways

  • dedup field removes duplicates, keeping the first occurrence
  • Use an integer (dedup 3 field) to keep multiple entries per value
  • Use sortby to control which duplicate is retained
  • Avoid dedup on _raw for performance reasons

Section 4.2

The sort Command

How sort Works

SyntaxDirectionNotes
sort + fieldnameAscending (A→Z, 0→9)Lowest/earliest first
sort - fieldnameDescending (Z→A, 9→0)Highest/latest first
sort 100 statusAscending (default)Limits to 100 results
sort 0 statusAll resultsNo limit (default cap = 10,000)

Alphanumeric fields sort lexicographically (dictionary order — uppercase before lowercase). Numeric fields sort by value. Always include a space after + or -.

🧪 Lab 4

Sort in Practice

  1. Sort ascending:
    SPL
    index=_internal sourcetype=splunkd | stats count by component | sort + count
  2. Sort descending:
    SPL
    index=_internal sourcetype=splunkd | stats count by component | sort - count
  3. Limit results:
    SPL
    index=_internal sourcetype=splunkd | stats count by component | sort 5 - count

Section 4.2

The eval Command

What Does eval Do?

eval calculates expressions and stores the result in a new or existing field. It supports math, strings, Boolean logic, and functions.

Syntax
eval new_field = expression

Three Expression Types

TypeOperatorsExample
Mathematical+ - * / %eval total = price * quantity
String. (concatenation)eval fullname = first . " " . last
BooleanAND OR NOT == != < >eval is_error = status != 200

Key Functions

FunctionPurposeExample
if(cond, true, false)Conditional logiceval status_label = if(status==200, "OK", "Problem")
case(c1,v1, c2,v2...)Multiple conditionseval sev = case(status==404,"Not Found", status==500,"Server Error")
lower(field)Convert to lowercaseeval clean = lower(username)
tostring(val, type)Format outputeval fmt = tostring(4056, "duration") → 01:07:36
pow(base, exp)Raise to powereval ips = pow(2,32)
strftime(time, fmt)Format timestampseval date = strftime(_time, "%m/%d/%Y")
cidrmatch(cidr, ip)IP range checkeval src = if(cidrmatch("10.0.0.0/8",ip), "local", "external")
⚠️ eval Syntax Rules
  • Field values are case-sensitive in eval comparisons
  • String values must use double quotes
  • Field names are unquoted (use single quotes only if name has special characters)
  • Use period (.) for concatenation, NOT plus (+)
  • eval changes are temporary — indexed data is never modified
🧪 Lab 5

eval — Create and Transform Fields

  1. Create a new field:
    SPL
    index=_internal sourcetype=splunkd | eval label = "Host: " . host | table label host component
  2. Use if() for conditional logic:
    SPL
    index=_internal sourcetype=splunkd | eval severity = if(log_level=="ERROR", "High", "Normal") | table _time host log_level severity
  3. Use case() for multiple conditions:
    SPL
    index=_internal sourcetype=splunkd | eval severity = case(log_level=="ERROR","High", log_level=="WARN","Medium", log_level=="INFO","Low") | table _time host log_level severity
🧪 Lab 6

eval — Math Functions

  1. Calculate IPv4 address space:
    SPL
    | makeresults | eval Total_IPv4 = tostring(pow(2,32), "commas")
  2. Format duration:
    SPL
    | makeresults | eval readable_time = tostring(4056, "duration")
  3. Convert to hex:
    SPL
    | makeresults | eval hex = tostring(4056, "hex"), decimal = 4056
  4. Format a UNIX timestamp:
    SPL
    | makeresults | eval epoch=1685461788 | eval formatted=strftime(epoch, "%H:%M:%S - %m/%d/%Y")

Lesson 200.4 Review

1. What character passes output between pipeline stages?▼
The pipe character |. It connects commands, passing results from one stage to the next.
2. What are the benefits of the fields command?▼
It focuses on specific fields, removes unnecessary data, and improves search performance by reducing data volume.
3. If you omit +/- from the fields command, what happens?▼
It includes the listed fields (positive filtering is the default).
4. What character separates multiple field names in the fields command?▼
A space. Example: fields host status method
5. What does the dedup command do?▼
Removes duplicate events based on specified field values, keeping only the first occurrence.
6. Which operator does eval use to concatenate strings?▼
The period/dot (.) operator. Example: eval fullName = firstName . " " . lastName
7. What types of expressions can eval evaluate?▼
Mathematical (price * quantity), Boolean (status == 200), Conditional (if/case), and String (lower, concatenation).
8. Name some functions supported by eval.▼
if(), case(), lower(), upper(), cidrmatch(), round(), len(), pow(), tostring(), strftime()
9. What is the default result limit for the sort command?▼
10,000 results. Use sort 0 to remove the limit.

Section 5.1

What Are Transforming Commands?

Transforming Commands Defined

Transforming commands reorganize raw search results into structured data tables. They convert event values into numbers, summaries, and aggregations — enabling charts, reports, and dashboards.

When you use a transforming command, your search becomes a transforming search — results appear on the Statistics and Visualization tabs rather than the Events tab.

Common Transforming Commands

CommandPurpose
topMost frequent values in a field
rareLeast frequent values in a field
statsSummary statistics (count, sum, avg, min, max...)
chartCreate data series for charts
timechartTime-based trends
🏆
Section 5.2

The top Command

How top Works

top returns the most frequent values in a field. By default, it shows the top 10 and adds count and percent columns automatically.

SPL
index=main sourcetype=access_combined | top clientip

top Options

OptionPurposeExample
limit=N or top NChange how many results (0 = all)top limit=5 status
countfield="name"Rename the count columntop clientip countfield="Hits"
percentfield="name"Rename the percent columntop clientip percentfield="Share"
showcount=falseHide count columntop status showcount=false
showperc=falseHide percent columntop status showperc=false
useother=trueAdd "OTHER" row for remaining valuestop limit=3 status useother=true
otherstr="label"Rename OTHER rowotherstr="Remaining"
by fieldGroup top values by another fieldtop clientip by host limit=3
🧪 Lab 7

Explore the top Command

  1. Default top 10:
    SPL
    index=_internal sourcetype=splunkd | top component
  2. Top 5 with custom column names:
    SPL
    index=_internal sourcetype=splunkd | top limit=5 component countfield="Occurrences" percentfield="Share %"
  3. Top by group with OTHER:
    SPL

  4. Hide count/percent:
    SPL
    index=_internal sourcetype=splunkd | top log_level showcount=false showperc=false

Section 5.3

The rare Command

How rare Works

rare is the opposite of top — it returns the least frequent values in a field. It supports the same options as top (limit, countfield, percentfield, showcount, showperc, useother, otherstr, by).

SPL
index=_internal sourcetype=splunkd | rare component
Security Use Case

In security operations, rare is invaluable for anomaly detection. Rare user agents, rare source IPs, rare URIs — these often indicate malicious activity that hides in the noise.

🧪 Lab 8

Find the Rare Values

    SPL
    index=_internal sourcetype=splunkd | rare component
  2. Rare with limit and custom label:
    SPL
    index=_internal sourcetype=splunkd | rare limit=5 component countfield="Times Seen" percentfield="Frequency %"
  3. Compare: Run top component and rare component side by side — what's the relationship?

Section 5.4

The stats Command

How stats Works

stats generates summary statistics from your data. Without a by clause, it returns a single summary row. With by, it returns one row per group.

SPL
index=main sourcetype=access_combined | stats count, sum(bytes) as "Total Bytes" by clientip

Core stats Functions

CategoryFunctionPurpose
Aggregatecount / count(field)Count all events / events with that field
dc(field)Distinct count — number of unique values
sum(field)Sum of numeric values
avg(field)Average value
min(field) / max(field)Minimum / Maximum value
median(field)Middle value
range(field)Difference between max and min
Event Orderfirst(field)First value (earliest event)
last(field)Last value (latest event)
Multivaluelist(field)All values (including duplicates)
values(field)Unique values only
Timeearliest(field)Earliest value
latest(field)Latest value

Knowledge Check

Lesson 200.5 Review

1. What are transforming commands?▼
Commands that take search results and transform them into structured data tables. They reorder, aggregate, and summarize data. Examples: stats, chart, timechart, top, rare.
2. What does the top command do?▼
Returns the most common values of a field in descending order of frequency.
3. How many values does top return by default?▼
10 values by default.
4. What keyword controls the number of results from rare?▼
limit. Example: rare limit=5 status.
5. How do you remove the percent column from rare results?▼
showperc=false. Example: rare showperc=false status.
6. What does stats dc() do?▼
Calculates the distinct count — the number of unique values for a field. Example: stats dc(clientip) counts unique IPs.
7. Why use the by clause with stats?▼
To group results by one or more fields. Without by, you get one summary row. With by, you get one row per group.
8. What 2-letter keyword renames a stats field?▼
as. Example: stats count as "Total Events".
